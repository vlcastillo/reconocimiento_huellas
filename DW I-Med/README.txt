DBA (BD de bonos electrónicos):
La tabla central es resumenbonos_AAAA recoge todas las transacciones (bonos emitidos) durante un año AAAA (hay una tabla por cada año desde el 2005). La transacción tiene un id numoperacion, y para cada numoperacion existe uno o más foliobono que representa el número de un bono electrónico emitido. La transacción tiene varios atributos que la describen, entre los más relevantes están: fecha de emisión (fecemision), financiador (codfinanciador / nombrefinanciador), lugar atención (codlugar / nombre lugar), prestador (rutconvenio / nombreprestador), datos del beneficiario (rutbeneficiario, nombrebeneficiario, edadbeneficiario, sexobeneficiario), seguro complementario (codseguro / nombreseguro) y desglose de costo y bonificaciones del bono (mtototal, aportefin, aporteseg, copagobenef).
Esta tabla no incluye las prestaciones asociadas a cada bono, por lo tanto, se debe recurrir a la tabla cfbonoprestacion_AAAA para obtener el codprestacion usando la llave {numoperacion, foliobono}. Luego en la tabla cfdetsolicitudvta_AAAA, usando {numoperacion, codprestacion} puedo recuperar la descripción de la prestación (desprestacion), la cantidad de esa prestación que se compro (cantidad), el monto total (mtototal) y el copago (mtocopago).

LME (BD de licencia médica electrónica):
La tabla central es lme_licencias, que recoge gran parte de los atributos de la licencia: datos trabajador (rut_trabajador, edad_trabajador, sexo_trabajador…), características de la licencia (dias_licencia, fecha_emision, fecha_inicio_reposo,…), datos del profesional médico (rut_profesional, pro_especialidad, …), datos del empleador (id_empleador, rut_empleador). Esta tabla tiene varias referencias a códigos (codigo_tipo_lcc, codigo_lugar_reposo,)..se incluyen dichas tablas que describen estos códigos.
